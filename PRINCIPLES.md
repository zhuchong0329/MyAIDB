# MyAIDB 原初准则

本文档记录 MyAIDB 已经对齐的基本原则。它不包含实现路线图、模块拆分、技术选型、里程碑或任务列表；后续任何设计和实现计划都应先引用并服从本文档。

## 项目定位

- 面向数据库学习者/研究者。
- 本地服务优先。
- 教学内核优先。
- 第一阶段以内存内核为主。

## SQL 原则

- SQLite 子集作为心智基线。
- 不承诺完整 SQLite 兼容。
- 类型系统极小化：`integer`、`real`、`text`、`blob`、`null`、`vector`。

## autoEmbed 原则

- autoEmbed 是声明式语义索引。
- 表数据遵守 ACID，embedding 索引是异步派生状态。
- embedding 索引状态必须可观察：`pending`、`ready`、`stale`、`error`。
- 本地模型优先。
- 模型采用外部资产登记。
- autoEmbed 是内核级扩展。

## 语义检索原则

- 当传统 SQL 确定性与语义检索体验冲突时，语义检索体验优先。
- 不确定性、过期状态、失败状态必须显式暴露。

## 非目标

- 第一阶段不追求分布式。
- 不追求完整 SQL 兼容。
- 不追求极致性能。
- 不做生产级持久化、WAL、checkpoint 或崩溃恢复。

## 设计气质

- 可读、可改、可解释。
- 模块边界清楚。
- 适合作为数据库内核与 AI 数据库能力的学习实验台。
